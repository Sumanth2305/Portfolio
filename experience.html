ChirpStack Configuration Experience

Overview

The objective of this document is to show the process of configuring the ChirpStack Network Server on both Ubuntu 22.04 which runs inside a Virtual Machine and on Windows OS. This process involves configuring the ChirpStack Network Server to which a gateway will be initialized. Subsequently, a device will be assigned to this gateway to facilitate uplink and downlink operations.

The data from these operations will be stored in an InfluxDB database which will be integrated into ChirpStack Network Server. This database is configured with Grafana to enable the visualization and analysis of the data on web pages. These Grafana dashboards are later embedded into an application, which will also be used to send downlinks to LoRa Devices through a UI.

LoRaWAN Specification

The LoRaWAN specification is a Low Power, Wide Area (LPWA) networking protocol designed to wirelessly transmit data of devices. LoRa devices are classified into different classes based on their power consumption and communication capabilities:

  • Class A: Devices send an uplink and “listen” for a message from the network one and two seconds after the uplink before going back to sleep.
  • Class B: Devices wake up and open a receive window to listen for a downlink according to a configurable, network-defined schedule.
  • Class C: Devices constantly listen for downlink messages from the network, except when transmitting data.

Gateways

Devices are connected to gateways to send data. Gateways serve as intermediary devices that receive and transmit data between LoRa devices and the internet. They can be classified into:

  • Indoor Gateways: Designed for use within buildings, featuring less robust enclosures and lacking weatherproofing.
  • Outdoor Gateways: Built to withstand outdoor conditions with sturdy, waterproof enclosures, often supporting Power over Ethernet (PoE).

ChirpStack Network Server

The LoRaWAN Network Server is at the core of every LoRaWAN Network, enabling connectivity, management, and monitoring of devices, gateways, and end-user applications. ChirpStack, an open-source LoRaWAN Network Server, is one such network server that allows the setup of private or public LoRaWAN networks. It provides:

  • A web interface for managing gateways, devices, and tenants.
  • Integration with databases, cloud providers, and services to handle device data.
  • A gRPC-based API for extending its capabilities.
